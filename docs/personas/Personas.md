
## Persona 1 — Henry (Commuter Student)

- **Primary user type**: Commuter
- **Frequency of use**: Daily (Weekday)
- **Background**: 23, university student in Dublin; commutes every day; often in a hurry.
- **Tech level**: High; uses mobile apps and maps a lot; fine with sharing location.
- **Context / Scenario**: Travels at peak times; checks station status while walking to a station.
- **Goals**
  - Find a station that has bikes, and a destination station that has free docks.
  - Avoid delays and reduce uncertainty.
  - Decide quickly with as few steps as possible.
- **Pain points**
  - Gets to a station and it’s empty or full.
  - Info can be old; no clear “last updated” time.
  - Too many stations to compare quickly.
- **Key needs (in scope)**
  - A map showing current occupancy and “last updated”.
  - Quick refresh and clear colors/legend.

## Persona 2 — Miguel (Tourist)

- **Primary user type**: Tourist / Short-term user
- **Frequency of use**: A few times during a trip
- **Background**: 29, tourist in Dublin for a few days; doesn’t know the station locations.
- **Tech level**: Medium; can use maps, but prefers simple apps.
- **Context / Scenario**: Makes spontaneous trips; wants fast, easy information.
- **Goals**
  - Find the nearest station that is usable.
  - Understand availability at a glance.
  - Check weather to decide whether to cycle.
- **Pain points**
  - Too many details; the UI can feel confusing.
  - Not sure which stations are a good choice.
  - Weather changes (rain/wind) make planning harder.
- **Key needs (in scope)**
  - A clear station popup with key numbers.
  - A simple weather widget and basic “good/bad to cycle” hint.

## Persona 3 — Siobhán (Operations / Data-Focused User)

- **Primary user type**: Operations / Data-focused stakeholder
- **Frequency of use**: Regular (e.g., daily/weekly checks)
- **Background**: 41, operations/data-focused user; wants reliable data and trends.
- **Tech level**: Medium; comfortable with dashboards and summaries.
- **Context / Scenario**: Checks the system for issues; looks for anomalies and patterns.
- **Goals**
  - Notice missing/stale data updates quickly.
  - Spot stations with extreme states (very empty/very full).
  - Use predictions to explain patterns.
- **Pain points**
  - Data issues are easy to miss.
  - A map alone is hard to filter and prioritize.
  - Without analytics, patterns are hard to explain.
- **Key needs (in scope)**
  - Clear data freshness indicators and simple filters/top lists.
  - Predictions shown in a clear, simple way.
